# Release Task Template

## Required Fields

- Version or release identifier.
- Included changes.
- Release checklist.
- Rollback plan.
- Required approvals.

## Lifecycle Emphasis

- In `planning`, identify validation, packaging, publishing, and communication
  steps.
- In `validation`, run the project's release checks and verify generated
  artifacts.
- In `delivery`, record published version, command evidence, and release notes.
- In `closed`, confirm follow-up monitoring or post-release checks.
