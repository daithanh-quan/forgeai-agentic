# Refactor Task Template

## Required Fields

- Current pain or risk.
- Behavior that must remain unchanged.
- Affected modules.
- Migration or compatibility constraints.

## Lifecycle Emphasis

- In `planning`, define non-goals and invariants.
- In `assignment`, keep write scopes narrow to reduce merge risk.
- In `validation`, prioritize regression tests and public API checks.
- In `review`, check for accidental behavior changes and unnecessary churn.
