# Bug Task Template

## Required Fields

- Observed behavior.
- Expected behavior.
- Reproduction steps.
- Affected version/environment.
- Suspected area.

## Lifecycle Emphasis

- In `triage`, classify severity and user impact.
- In `planning`, identify the smallest fix and regression test.
- In `validation`, prove the reproduction fails before the fix when practical,
  then passes after the fix.
- In `memory-update`, record recurring bug patterns or prevention rules only
  when they are likely to recur.
