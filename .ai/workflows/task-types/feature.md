# Feature Task Template

## Required Fields

- User or system actor.
- Problem statement.
- Desired behavior.
- Acceptance criteria.
- Rollout or compatibility constraints.

## Lifecycle Emphasis

- In `triage`, decide whether OpenSpec is required.
- In `planning`, define in-scope and out-of-scope behavior explicitly.
- In `specification`, update proposal, design, tasks, and capability spec when
  behavior or contracts change.
- In `validation`, include both automated checks and user-flow/manual checks
  when the feature has UI or workflow impact.
