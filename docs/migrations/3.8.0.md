# Migration Guide ‚Äî 3.8.0

## What changed

Phase 12B completes the LLM-native adapter layer. This is an **additive**
release ‚Äî no breaking config or schema changes.

- **Streaming output.** `--route --adapter <name> --stream` streams model output
  to stdout incrementally. Without `--stream`, the buffered 3.7.0 behavior is
  unchanged. CLI adapters ignore `--stream`.

- **Retry with backoff.** Two new optional per-adapter fields in
  `.ai/api-adapters.json`:
  - `max_retries` ‚Äî integer 0‚Äì5, default `2`. `0` disables retry.
  - `retry_base_ms` ‚Äî positive integer, default `500`. Backoff is
    `retry_base_ms * 2^attempt`.
  Retryable failures (network, 5xx, 429) are retried before the existing
  quota‚ÜíCLI fallback. Auth and mid-stream failures are never retried.

- **Lifecycle events.** `run_start`, `retry_attempt`, and `run_complete` are
  emitted as NDJSON to the `--watch` pipe (best-effort) and rendered in the
  activity log.

- **`retry_count` on run records.** Run records written before 3.8.0 do not have
  this field; they are read as `retry_count: 0`. No manual action needed.

## How to migrate

```bash
forgeai-init --upgrade
```

This refreshes the `.ai/api-adapters.json` template (adding the documented
`max_retries` / `retry_base_ms` example fields) without overwriting a customized
config. Existing configs keep working unchanged ‚Äî the new fields are optional and
default to `2` / `500`.

No changes are required to existing run records, compiled context artifacts, or
CLI adapters.
